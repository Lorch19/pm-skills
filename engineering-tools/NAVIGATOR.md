# Engineering Tools — Navigator

## Quick Decision Tree

### Build & Ship
- Architecture decision or system design → `architecture`
- Pre-deployment verification → `deploy-checklist`
- Writing technical docs → `documentation`
- Test strategy for a feature/system → `testing-strategy`

### Review & Improve
- Reviewing code changes → `code-review`
- Identifying and prioritizing tech debt → `tech-debt`

### Debug & Respond
- Debugging an issue → `debug`
- Production incident → `incident-response`

### Daily Workflow
- Preparing standup update → `standup`
- Designing a new system → `system-design`

## All Skills

| Skill | Use When |
|-------|----------|
| `architecture` | ADRs, technology choices, design evaluation |
| `code-review` | PR review for security, performance, correctness |
| `debug` | Structured debugging (reproduce, isolate, diagnose, fix) |
| `deploy-checklist` | Pre/during/post deployment verification |
| `documentation` | READMEs, API docs, runbooks, onboarding guides |
| `incident-response` | Triage, communicate, mitigate, postmortem |
| `standup` | Daily standup generation from recent activity |
| `system-design` | Requirements, high-level design, scalability |
| `tech-debt` | Categorize and prioritize technical debt |
| `testing-strategy` | Test plans, coverage, testing pyramid |
