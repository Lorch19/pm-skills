# System Design

Help design systems and evaluate architectural decisions.

## Framework

### 1. Requirements Gathering
- Functional requirements (what it does)
- Non-functional requirements (scale, latency, availability, cost)
- Constraints (team size, timeline, existing stack)

### 2. High-Level Design
- Component diagram, data flow, API contracts, storage choices

### 3. Deep Dive
- Data model, API endpoints, caching, queues, error handling

### 4. Scale and Reliability
- Load estimation, horizontal vs. vertical scaling, failover, monitoring

### 5. Trade-off Analysis
- Every decision has trade-offs. Make them explicit.
- Consider: complexity, cost, team familiarity, time to market, maintainability

## Output

Clear, structured design documents with diagrams, explicit assumptions, and trade-off analysis. Always identify what to revisit as the system grows.
