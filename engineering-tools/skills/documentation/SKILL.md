# Technical Documentation

Write clear, maintainable technical documentation for different audiences.

## Document Types

### README
- What this is and why it exists
- Quick start (< 5 minutes to first success)
- Configuration and usage

### API Documentation
- Endpoint reference with request/response examples
- Authentication, error codes, rate limits

### Architecture Doc
- Context and goals, high-level design with diagrams
- Key decisions and trade-offs, data flow

### Onboarding Guide
- Environment setup, key systems
- Common tasks with walkthroughs

## Principles

1. **Write for the reader** — Who is reading and what do they need?
2. **Start with the most useful information**
3. **Show, don't tell** — Code examples, commands, screenshots
4. **Keep it current** — Outdated docs are worse than no docs
5. **Link, don't duplicate**
