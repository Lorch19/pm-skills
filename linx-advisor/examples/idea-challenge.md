# Example: Idea Challenge

**Omri:** "I think we should build JIT access next — it's a hot feature and Opal is getting traction."

**Advisor response:**

Pushing back on this one.

**Competitive check:** JIT is Opal's strength (they lead the category), but Opal scores 72/100 overall — lowest weighted score among competitors. JIT is a feature, not a platform differentiator. ConductorOne and Lumos also have JIT. Building JIT doesn't widen Linx's moat.

**Positioning check:** JIT doesn't strengthen the "only we" claim. It's table stakes for modern IGA, not identity coverage or autonomous remediation.

**What the data says you should build instead:**
- Access reviews/certification (Absent today, every competitor has it, blocks enterprise sales)
- Governance & Policy V1.1 (already Building — ship it before starting something new)
- Compliance reporting (required for regulated verticals, strengthens Series B narrative)

**The priority framework:**
1. Ship what's Building (Governance V1.1)
2. Close the gaps that block enterprise deals (access reviews, compliance)
3. Then add growth features (JIT, provisioning)

JIT is a "nice to have" — access reviews are "can't close the deal without it."

**What would have to be true for JIT to be the right priority?** You'd need evidence that current deals are being lost specifically because of missing JIT (not governance or compliance). Do you have that signal from sales?
